# AXIOMS

## AX-MOD-CLI-001: Agent-host lockfile schema is a cross-language contract
The `AgentHostMetadata` lockfile schema in `src/tunnels/agent_host_metadata.rs` is
shared verbatim (camelCase field names) with the TypeScript SSH client at
`src/vs/platform/agentHost/common/remoteAgentHostMetadata.ts`; fields must not be
renamed or removed, and incompatible changes must bump
`AGENT_HOST_METADATA_SCHEMA_VERSION` and keep older lockfiles parseable (optional
fields degrade to `None`).

### Applies To
- `cli/src/tunnels/agent_host_metadata.rs`
- `cli/src/commands/agent_host.rs` (lockfile readers/writers)
- contract mirror: `src/vs/platform/agentHost/common/remoteAgentHostMetadata.ts`

### Acceptance Checks
- `grep -q 'rename_all = "camelCase"' cli/src/tunnels/agent_host_metadata.rs && grep -q 'AGENT_HOST_METADATA_SCHEMA_VERSION' cli/src/tunnels/agent_host_metadata.rs && test -f src/vs/platform/agentHost/common/remoteAgentHostMetadata.ts` (camelCase contract + schema-version constant present, TS mirror exists)
- `cd cli && cargo test agent_host` (lockfile round-trip + token tests pass)
- manual: any field change in `AgentHostMetadata` is mirrored in `remoteAgentHostMetadata.ts` within the same change; `serde(skip_serializing_if = "Option::is_none")` is preserved on optional fields.

### Enforcement
computational

## AX-MOD-CLI-002: Supervisor daemonization handshake constants are frozen
The foreground↔supervisor handshake depends on the exact values of
`SUPERVISOR_ENV` (`VSCODE_AGENT_HOST_SUPERVISOR`) and the stdout sentinel
`SUPERVISOR_READY_LINE` (`__VSCODE_AGENT_HOST_READY__`); these must not change
without updating both the spawner and the watcher in the same change, because a
mismatch silently hangs `code agent host` until `SUPERVISOR_READY_TIMEOUT`.

### Applies To
- `cli/src/commands/agent_host.rs` (`run_supervisor`, `daemonize_supervisor`, `ensure_supervisor_running`)

### Acceptance Checks
- `grep -q 'VSCODE_AGENT_HOST_SUPERVISOR' cli/src/commands/agent_host.rs && grep -q '__VSCODE_AGENT_HOST_READY__' cli/src/commands/agent_host.rs` (both handshake constants present in the single file that spawns and watches)
- `cd cli && cargo build` (compiles)
- manual: `SUPERVISOR_ENV` set on spawn is the same string checked in `agent_host()`; the line printed by the supervisor equals the line matched by the foreground/`ensure_supervisor_running` loops.

### Enforcement
computational

## AX-MOD-CLI-003: AHP client requests must transparently recover from AuthRequired
Agent client commands (`agent ps|stop|logs`) must issue AHP requests through
`commands::agent::request_with_auth`, which on an `AUTH_REQUIRED`
(`ahp_error_codes::AUTH_REQUIRED`, `-32007`) error runs the device-flow login and
retries the request exactly once; commands must not call `client.request` directly
for auth-gated methods or duplicate this retry logic.

### Applies To
- `cli/src/commands/agent.rs`
- `cli/src/commands/agent_ps.rs`
- `cli/src/commands/agent_stop.rs`
- `cli/src/commands/agent_logs.rs`

### Acceptance Checks
- `grep -q 'request_with_auth' cli/src/commands/agent_ps.rs cli/src/commands/agent_stop.rs cli/src/commands/agent_logs.rs` (ps/stop/logs route through the helper)
- `! grep -qE 'client\.request' cli/src/commands/agent_ps.rs cli/src/commands/agent_stop.rs cli/src/commands/agent_logs.rs` (no raw `client.request` in the auth-gated client commands)
- `cd cli && cargo build` (compiles)

### Enforcement
computational

## AX-MOD-CLI-004: The clap command surface is the public CLI contract
The command/subcommand/flag set defined in `src/commands/args.rs` (including the
`agent host|ps|stop|kill|logs` family and the legacy-argv translation in
`bin/code/legacy_args.rs`) is a user- and script-facing contract; renaming or
removing a command, subcommand, or flag is a breaking change requiring a governed
task with help-text and legacy-arg updates in the same change.

### Applies To
- `cli/src/commands/args.rs`
- `cli/src/bin/code/main.rs`
- `cli/src/bin/code/legacy_args.rs`

### Acceptance Checks
- `grep -q 'enum AgentSubcommand' cli/src/commands/args.rs && grep -qE 'Host\(AgentHostArgs\)' cli/src/commands/args.rs` (the `agent` subcommand surface is intact)
- `cd cli && cargo build` (compiles; clap derive validates the surface)
- manual: removed/renamed flags are reflected in `legacy_args.rs` translation and any referencing docs.

### Enforcement
computational

## AX-MOD-CLI-005: Connection-token files are written with owner-only permissions
Minted agent-host connection tokens (`mint_connection_token`) and any
secret-bearing files must be created with restrictive (`0o600`) permissions on
Unix and persisted atomically; weakening these permissions or logging token values
is forbidden.

### Applies To
- `cli/src/commands/agent_host.rs` (`mint_connection_token`)
- `cli/src/tunnels/agent_host_metadata.rs` (lockfile write / dir `0o700`)

### Acceptance Checks
- `grep -q '0o600' cli/src/tunnels/agent_host_metadata.rs && grep -q '0o700' cli/src/tunnels/agent_host_metadata.rs` (owner-only file + dir modes retained)
- `cd cli && cargo test agent_host` (token mint/persist/override + `writes_owner_only_permissions` tests pass)
- manual: `grep -n "0o600\|0o700" cli/src/commands/agent_host.rs cli/src/tunnels/agent_host_metadata.rs` confirms owner-only modes are retained and no token value is logged.

### Enforcement
computational
